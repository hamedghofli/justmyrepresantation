1: مقدمه

ORM (Object-Relational Mapping) به برنامه‌نویسان اجازه می‌دهد با پایگاه داده به‌صورت شیء‌گرا تعامل کنند.

Peewee یک ORM سبک و ساده برای Python است که برای کار با پایگاه داده‌های مختلف استفاده می‌شود.

2: مزایای استفاده از orm ها

کاهش پیچیدگی در نوشتن کد SQL خام.

ارتباط مستقیم بین جداول پایگاه داده و مدل‌های برنامه.

نگهداری و توسعه آسان‌تر کد.

جلوگیری از مشکلات امنیتی مانند SQL Injection.

3: مزایای استفاده از Peewee

سبک و سریع: مناسب برای پروژه‌های کوچک تا متوسط.

سادگی در یادگیری: مستندات خوب و پیاده‌سازی آسان.

پشتیبانی از پایگاه داده‌های : SQLite، MySQL، PostgreSQL و ...

امکانات کامل: پشتیبانی از Query Building، Migrations، Joins و ...

4: نصب Peewee


pip install peewee

5: تعریف یک مدل در Peewee

from peewee import *
db = SqliteDatabase('example.db')
class BaseModel(Model):
class Meta:
database = db

class User(BaseModel):
username = CharField(unique=True)
email = CharField()
age = IntegerField()
              
db.connect()
db.create_tables([User])

6: ایجاد و مدیریت رکوردها

ایجاد رکورد:

user = User.create(username='john_doe', email='john@example.com', age=30)

بازیابی داده‌ها:

users = User.select()
for user in users:
print(user.username, user.age)

به‌روزرسانی رکورد:

user = User.get(User.username == 'john_doe')
user.age = 31
user.save()

حذف رکورد:

user.delete_instance()

7: استفاده از Queries پیچیده

# فیلتر کردن کاربران با سن بیشتر از 25
users = User.select().where(User.age > 25)

# مرتب‌سازی کاربران بر اساس سن
users = User.select().order_by(User.age.desc())

# استفاده از Join
class Blog(BaseModel):
    title = CharField()
    user = ForeignKeyField(User, backref='blogs')

blogs = Blog.select().join(User).where(User.username == 'john_doe')

8: مدیریت Migration‌ها

Peewee ابزار playhouse.migrate را برای مدیریت تغییرات پایگاه داده فراهم می‌کند.

from playhouse.migrate import *

migrator = SqliteMigrator(db)
migrate(
    migrator.add_column('user', 'birth_date', DateField(null=True)),
    migrator.drop_column('user', 'age')
)

10: نتیجه‌گیری

Peewee یک انتخاب مناسب برای پروژه‌هایی است که به یک ORM ساده و سریع نیاز دارند.

امکانات مناسب آن نیازهای عمومی در کار با پایگاه داده را برطرف می‌کند.

برای پروژه‌های بزرگ‌تر، ممکن است گزینه‌های پیچیده‌تر مناسب‌تر باشند.
